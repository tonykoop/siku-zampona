# Siku (Zampoña) — Images

**Refs:** https://github.com/tonykoop/siku-zampona/issues/1
Generated: 2026-05-19

This folder is a placeholder. No hero photo, concept render, or build photo
lives here yet.

## Photo Plan

The authoritative shot list — what to shoot, in what order, with which
neutral background and which scale reference — lives in
**`photo-shotlist.md`** at the repo root. Read that file before adding any
file to this folder.

`photo-shotlist.md` defines groups RAW, CUT, ASM, FIN, PLAY, and HERO with
explicit shot IDs (RAW-01, CUT-03, etc.). When a real image lands here, it
should be filed under the shot ID, e.g. `images/RAW-01-bamboo-culm.jpg`.

## What Belongs Here

- **Build photos** taken during a real Zanka or family-member build,
  matching shot IDs from `photo-shotlist.md`.
- **Hero photo** of a finished instrument, owned by the photographer with
  licence recorded.
- **Concept renders** clearly framed as concept-only per
  `visual-bom-brief.md`. File these alongside a sibling
  `<filename>.provenance.md` that records the generator (Wolfram, AI tool,
  hand-illustration), the prompt or notebook cell, and the licence
  posture.

## What Does Not Belong Here

- **Native CAD files** — those belong in `cad/`.
- **Dimensioned drawings** — those belong in `drawings/`.
- **Anonymous AI-generated images with no provenance record** — see
  `visual-bom-brief.md` and the v4.3 provenance convention; an image
  without a sibling provenance note must not land here.
- **Photos of historical or museum instruments** without documented
  permission or a clear public-domain trace.

## Honesty Rules

- Until a real photo lands here, `README.md` must not show a hero image
  claiming to depict a built siku from this repo.
- A render of a CAD model is a render, not a photo. Caption accordingly.
- Photos taken during another maker's build must record consent and the
  maker's name.
